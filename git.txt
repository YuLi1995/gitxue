 什么是Get？
	分布式版本控制系统
Get的诞生
	Linux linus
分布式和集中式
设置名字 邮箱
	$ git config --global user.name ""
	$ git config --global user.email ""
创建版本库
	创建一个空目录，作为版本库
	初始化版本库
		1 进入到版本库所在目录
		2 执行初始化命令
			$ git init
把文件添加到版本库
	1 创建一个文件
	2 在仓库里面查看状态
		$ git status
	3 把文件添加到版本库
		$ git add 文件名
	4 把文件提交给版本库
		$ git commit -m "信息"
	5 查看文件每个版本
		$ git log
时光穿梭
	回到以前的版本
	$ git reset --hard 版本
	版本:
		HEAD:代表当前版本
		HEAD^:代表上一个版本
		HEAD^^：代表上上个版本
		HEAD~100:代表上100个版本
	后悔药：
		git reflog:查看历史所有版本信息
修改撤销：
	$ git checkout -- 文件名
场景：
	1 在提交之前，撤销修改
		$ git checkout -- 文件名
	2 在文件已提交到暂存区，撤销修改
		清除暂存区文件
			$ git reset HEAD 文件名
		替换工作区文件
			$ git cheakout -- 文件名
	3 在文件已经被commit之后，撤销修改
		版本退回
			$ git reset --hard 版本
删除文件
	不想删
		$ git cheakout -- 文件名
	“真”想删
		1 删除源文件
		2添加暂存区
			$ git add 文件名
		3 提交
			$ git commit -m "信息"
